Proyecto de To-Do List con Conexión a JSON:

1. Objetivo del Proyecto:

Desarrollar una aplicación de lista de tareas (To-Do List) que permita agregar, eliminar, completar y gestionar tareas de estudio.
2. Tecnologías Utilizadas:

HTML: Define la estructura del documento web.
CSS: Estiliza y da formato a la interfaz de usuario.
JavaScript: Maneja la lógica de la aplicación, interactúa con el DOM y realiza operaciones asincrónicas.
JSON: Almacena y recupera datos de las tareas.
3. Funcionalidades del Proyecto:

Inicio de Sesión:

Verifica el estado de inicio de sesión del usuario utilizando localStorage.
Permite al usuario iniciar sesión con un nombre de usuario y contraseña.
Lista de Tareas:

Permite al usuario agregar nuevas tareas de estudio.
Cada tarea tiene un nombre y un tiempo estimado para completarla.
Las tareas se pueden marcar como completadas, mover a la lista de tareas completadas o eliminar.
Total de Tiempo Estimado:

Muestra el tiempo total estimado para completar todas las tareas no completadas.
Persistencia de Datos:

Utiliza localStorage para almacenar información sobre el estado de inicio de sesión y las tareas del usuario.
Guarda y carga las tareas desde y hacia un archivo JSON (tareas.json) en el servidor.
Estilo Minimalista y Moderno:

Utiliza CSS para dar a la aplicación un estilo moderno y minimalista, facilitando la experiencia del usuario.
Proyecto Centrado en el Estudio:

Diseñado especialmente para estudiantes, como tú, que utilizan listas de tareas para organizar su estudio diario.
Asincronía y Operaciones Asincrónicas:

Implementa funciones asincrónicas para cargar y guardar datos, asegurando una experiencia de usuario fluida.
Usa Promesas y fetch para realizar operaciones asincrónicas.
4. Experiencia del Usuario:

Inicio de Sesión Simple:
Facilita el acceso a la aplicación mediante un proceso sencillo de inicio de sesión.
Gestión Intuitiva de Tareas:
Permite realizar acciones comunes de gestión de tareas de manera fácil e intuitiva.
Seguimiento del Tiempo:
Ayuda a los estudiantes a realizar un seguimiento eficiente del tiempo dedicado a cada tarea.
5. Contexto Personal:

Motivación del Proyecto:
La inspiración para este proyecto surge de la necesidad personal de organizar y gestionar tareas de estudio diarias.
Como estudiante de profesorado de historia, encuentras útil tener una herramienta personalizada que se adapte a tus necesidades de estudio.
6. Utilidad para el Estudio de Historia:

Organización de Tareas de Estudio:
Facilita la organización de tareas específicas relacionadas con el estudio de historia.
Permite gestionar eficientemente el tiempo dedicado a cada tarea de estudio.
7. Json Personalizado
La decisión de utilizar un JSON personalizado (tareas.json) como almacén de datos se tomó debido a la falta de disponibilidad de APIs externas específicas para las necesidades del proyecto. Esta elección brinda flexibilidad, control y estabilidad en la gestión de datos, permitiendo adaptar la aplicación de To-Do List a las necesidades específicas de un estudiante de profesorado de historia.
En Resumen:

Este proyecto de To-Do List se centra en proporcionar una herramienta simple y eficaz para organizar y gestionar las tareas de estudio diarias. La conexión con JSON y las operaciones asincrónicas mejoran la persistencia de los datos, garantizando que las tareas se guarden y carguen de manera eficiente. El diseño minimalista y moderno, junto con la facilidad de uso, crea una experiencia adaptada a las necesidades de un estudiante de profesorado de historia.
